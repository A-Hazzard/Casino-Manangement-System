## 1. Page Overview

Specialized tool for correcting meter gaps or audit discrepancies in historical data.

---

## 2. Data & API Architecture (By Section)

### 🌉 Gap Analysis

| Data Element         | Source API                      | Trigger                    |
| :------------------- | :------------------------------ | :------------------------- |
| **Timeline Gaps**    | `GET /api/collection-reports`   | Page Load / Select Machine |
| **Last Known Meter** | `GET /api/cabinets/[cabinetId]` | Select Machine             |

- **Filters Applied**: `machineId`, `timeRange`.
- **Implementation**: Logic on the backend identifies discontinuities in the `CollectionMetersHistory` sub-document.

### 🛠️ Bridge Calculator

| Data Element        | Source API                    | Trigger           |
| :------------------ | :---------------------------- | :---------------- |
| **Estimated Delta** | `GET /api/collection-reports` | Input "New Meter" |
| **Preview Result**  | `Computed`                    | Instant           |

- **Logic**: Backend calculates the `average daily movement` based on the surrounding 30 days of data to propose a "Bridge" entry.

---

## 3. High-Risk Action Triggers

- **Apply Fix**: Triggers `POST /api/collection-reports/fix-report`.
- **Force Resync**: Triggers `PATCH /api/cabinets/[cabinetId]/meters` (Updates the current snapshot).

---

## 4. Role Detection

- **Developer Only**: This entire route is protected by a server-side redirect if the `user.role` !== 'Developer'.
- **Audit Logging**: Every single click on this page generates a `SUPER-USER-ACTION` log entry.

---

## 5. Visual Indicators

- 🔴 **Gap**: Red vertical bar in the timeline indicating 0 movement or missing records.
- 🚧 **Bridge**: Amber badge indicating an entry that was generated by the tool rather than hardware.
