Reports Module Meeting Summary
Key Concepts & Definitions

Hold: Refers to the theoretical percentage of wagers a machine is expected to keep as profit over time.
Gaming Day: The defined operating hours for a location (start and end time), used to ensure accurate daily reporting.
Machine Types: Machines must be categorized during creation (e.g., as a roulette or slot machine) to allow for filtering in reports.
UI Color Coding: For Net Win/Revenue fields, the proposed color scheme is: Gross (Green), Drop (Yellow), and Total Cancelled Credits (Black).
Report & Feature Breakdown

Events Browser:
Its main purpose is for troubleshooting and maintaining compliance with Gaming Laboratories International (GLI) standards.
It will include a "Hide No Activity" filter to simplify the view.
Meters Export Report:
Used to export raw, detailed machine data like Coin In, Coin Out, Jackpot, and Games Played for a specific location over a chosen time period.


Comparison Report:
Designed to compare a machine's theoretical earnings against its actual real-world performance.
A key system constraint is that not all machine types can provide a theoretical hold value.
Location Evaluation Report:
Primarily used to track the performance of SAS machines, which provide immediate access to soft meter data.
Location Revenue Report:
Used to track the performance of non-SAS machines at a location, focusing on metrics like drop, cancelled credits, and gross.
This report includes graphs for visual analysis and shows the location's top-performing machines.
Machines Offline Report:
This tool shows which machines are currently offline and provides functionality for authorized users to edit machine values.
New Feature Requests

Automated Reporting: Aaron will investigate building a feature that allows users to schedule reports to be automatically generated and sent to a list of emails at a specific time.
